Routing

We use the Angular Router to manage navigation between views in our SPA

The router enables navigation by interpreting a browser URL as an instruction to change the view.

Defining routes

Routes can be defined in a routing module using RouterModule

In the routing module we will define our routes. After that we can import the routing module into our application module.

{{routingModule}}

We can use ** wildcard to match undefined routes. This allows use to display things like a "Page not found" message.

The order of routes is important because the Router uses a first-match wins strategy when matching routes.

Managing route access

We can control access to routes using route guards. These are functions that return true or false to allow or prevent navigation to a route based on a condition

Angular provides the following route guards

  • CanActivate — navigation to a route
  • CanActivateChild — navigation to a child route
  • CanDeactivate — navigation away from the current route
  • CanMatch — whether a Route should be used at all, even if the path matches the URL segment
  • resolve — perform route data retrieval before route activation
{{routeGuards}}

Navigation

Use routerLink to allow navigation to a route. This can be used in place of an href tag.

When the link is clicked the router will render the component on the defined route to the nearest router-outlet
